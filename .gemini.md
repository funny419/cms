# CMS 프로젝트 분석 보고서

## 개요
이 문서는 프로젝트 내 `.gitignore` 파일을 기반으로 **cms** 프로젝트의 기술 스택과 개발 환경을 분석한 내용을 담고 있습니다.

## 기술 스택 (Tech Stack)
### 백엔드 (Backend)
- **언어/프레임워크**: Flask (Port 5000)
- **데이터베이스**: MariaDB (Host Port: 4807, Container Port: 3306)
- **ORM/드라이버**: SQLAlchemy with `pymysql` driver.
  - **연결 문자열 형식**: `mysql+pymysql://<user>:<password>@<host>:<port>/<dbname>`

### 프론트엔드 (Frontend)
- **라이브러리/프레임워크**: React (Vite), Tailwind CSS
- **API 통신**: Axios (Target Port: 5000)
- **개발 서버**: Port 5173

### 패키지 관리 및 도구
- **Python**: `pip` + `venv`
- **Node.js**: `npm` or `yarn`

## 개발 환경 (Development Environment)
- **IDE**: Visual Studio Code
- **Infrastructure**: Docker Compose

### 프로젝트 구조 (Project Structure)
```text
cms/
├── backend/           # Flask Application (Port 5000)
│   ├── venv/          # (Excluded from analysis)
│   └── ...
├── frontend/          # React + Vite Application (Port 5173)
├── docker-compose.yml
└── .gitignore
```

## Coding Rules
1. Backend: 모든 API 응답은 JSON 형식을 유지할 것.
2. Frontend: 컴포넌트는 함수형(Functional)으로 작성하고, API 통신은 Axios를 사용할 것.
3. Database: MariaDB 내부 포트는 3306, 호스트 포트는 4807로 고정.
4. CORS: http://localhost:5173(Vite)의 접근을 항상 허용할 것.

## Special Instructions
- 생성형 AI 기능을 추가할 때는 Gemini API(Google Generative AI)를 우선적으로 고려할 것.
- 가상환경(venv) 경로인 `backend/venv/`는 분석에서 제외할 것.

## Core Principles (Adapted from Claude Tips)
- **Think First:** Always outline the plan in a `<thought>` block before writing code.
- **Stay Concise:** Avoid boilerplate; focus on clean, modular Python and React code.
- **Reference Context:** Use `@frontend` and `@backend` symbols to maintain consistency across the decoupled architecture.

## Safety & Quality
- No hardcoded secrets. Use `.env` files for all sensitive data.
- Ensure all React components are properly typed or use clear prop-types logic.
